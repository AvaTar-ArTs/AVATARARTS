# Steven's WordPress Blog

A WordPress blog setup using Docker for easy development and deployment.

## Quick Start

1. **Start the containers:**
   ```bash
   docker-compose up -d
   ```

2. **Access your blog:**
   - WordPress: http://localhost:8080
   - phpMyAdmin: http://localhost:8081

3. **Complete WordPress setup:**
   - Follow the WordPress installation wizard
   - Create your admin account
   - Choose a theme

## Features

- **WordPress Latest**: Always up-to-date WordPress installation
- **MySQL Database**: Reliable database backend
- **phpMyAdmin**: Easy database management
- **Persistent Data**: Your content survives container restarts
- **Custom Themes**: Place themes in `wp-content/themes/`
- **Custom Plugins**: Place plugins in `wp-content/plugins/`

## Directory Structure

```
wordpress-blog/
├── docker-compose.yml    # Docker configuration
├── wp-content/          # Custom themes and plugins
│   ├── themes/          # Custom themes
│   └── plugins/         # Custom plugins
└── README.md           # This file
```

## Customization

### Adding Custom Themes

1. Create a new theme directory in `wp-content/themes/`
2. Add your theme files (style.css, index.php, etc.)
3. Activate the theme in WordPress admin

### Adding Custom Plugins

1. Place plugin files in `wp-content/plugins/`
2. Activate plugins in WordPress admin

### Environment Variables

You can customize the setup by modifying the environment variables in `docker-compose.yml`:

- `WORDPRESS_DB_NAME`: Database name
- `WORDPRESS_DB_USER`: Database user
- `WORDPRESS_DB_PASSWORD`: Database password
- `MYSQL_ROOT_PASSWORD`: MySQL root password

## Backup and Restore

### Backup
```bash
# Backup database
docker exec steven-wordpress-db mysqldump -u wordpress -pwordpress_password steven_blog > backup.sql

# Backup WordPress files
docker cp steven-wordpress:/var/www/html ./wordpress-backup
```

### Restore
```bash
# Restore database
docker exec -i steven-wordpress-db mysql -u wordpress -pwordpress_password steven_blog < backup.sql

# Restore WordPress files
docker cp ./wordpress-backup steven-wordpress:/var/www/html
```

## Stopping and Cleanup

```bash
# Stop containers
docker-compose down

# Stop and remove all data
docker-compose down -v
```

## Production Deployment

For production deployment:

1. Use a production-ready database (not the included MySQL)
2. Set up proper SSL certificates
3. Configure a reverse proxy (nginx/Apache)
4. Set up regular backups
5. Use environment variables for sensitive data

## Troubleshooting

- **Port conflicts**: Change ports in docker-compose.yml
- **Permission issues**: Check file permissions in wp-content
- **Database connection**: Verify database credentials
- **Plugin conflicts**: Deactivate plugins one by one
