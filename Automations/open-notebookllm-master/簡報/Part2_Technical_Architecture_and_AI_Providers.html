üîå AI Provider Abstraction

All AI providers implement a unified interface, so switching providers is seamless:

class BaseAIProvider:
    def generate(self, prompt: str) -> str:
        """Generate text response"""
        raise NotImplementedError

    def generate_stream(self, prompt: str) -> Generator:
        """Stream response generation"""
        raise NotImplementedError

    def generate_embedding(self, text: str) -> List[float]:
        """Generate vector embeddings"""
        raise NotImplementedError

    def generate_image(self, prompt: str) -> str:
        """Generate image (base64)"""
        raise NotImplementedError
Design pattern: Strategy Pattern, so the app layer doesn't need to know which AI is used.

üîç RAG (Retrieval-Augmented Generation)

RAG is a core capability of this project:

1Ô∏è‚É£ Document processing

Upload files -> parse content -> split into chunks

2Ô∏è‚É£ Vector embeddings

Convert each chunk to vectors and store in the database

3Ô∏è‚É£ Retrieval

User question -> vectorize -> similarity search

4Ô∏è‚É£ Augmented generation

Send relevant chunks + question to the LLM for the answer

Benefit: AI answers are grounded in your data, reducing hallucinations and improving accuracy.

üöÄ Advanced RAG Search Techniques

RRF Fusion

Reciprocal Rank Fusion combines rankings from full-text and vector search.

Query Expansion

LLM expands the query with synonyms and related terms to improve recall.

LLM Reranking

LLM reorders results to improve precision.

Deduplication

Automatically merges duplicate results to avoid redundancy.
